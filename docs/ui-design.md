# VisuTask 前端界面设计

> 技术栈：React 18 + TypeScript + shadcn/ui + Tailwind CSS + Lucide Icons
> 框架：Wails v3（系统 WebView）

---

## 一、窗口规格

| 属性 | 值 |
|------|-----|
| 默认尺寸 | 1024 × 768 px |
| 最小尺寸 | 800 × 600 px |
| 宽高比 | 4:3 |

---

## 二、全局结构

无侧边栏设计，首页功能卡片作为所有模块入口，子页面通过面包屑导航返回。

```
┌──────────────────────────────────────────────────────┐
│  Header                                  h = 56 px   │
├──────────────────────────────────────────────────────┤
│                                                      │
│                 Content Area                         │
│                                                      │
│                                                      │
│                                                      │
├──────────────────────────────────────────────────────┤
│  StatusBar                               h = 28 px   │
└──────────────────────────────────────────────────────┘
```

| 区域 | 标准 (1024×768) | 最小 (800×600) |
|------|----------------|----------------|
| Header | 1024 × 56 | 800 × 56 |
| Content | 1024 × 684 | 800 × 516 |
| StatusBar | 1024 × 28 | 800 × 28 |

### 2.1 Header

```
┌──────────────────────────────────────────────────────┐
│  [Logo] VisuTask                    [⚙] [👤] VIP1    │
└──────────────────────────────────────────────────────┘
```

| 元素 | 位置 | 说明 |
|------|------|------|
| Logo + 应用名 | 左 | 36×36 图标 + "VisuTask" 文字 |
| 设置 | 右 | `Settings` 图标，跳转设置页 |
| 登录 / 头像 | 右 | 未登录显示"登录"文字，已登录显示圆形头像 |
| VIP 等级 | 右 | 头像旁 `Badge` 显示等级，普通用户不显示 |

三种状态：

```
未登录:    [Logo] VisuTask                          [⚙] [登录]
普通用户:  [Logo] VisuTask                          [⚙] [👤]
VIP 用户:  [Logo] VisuTask                          [⚙] [👤] VIP1
```

### 2.2 StatusBar

```
┌──────────────────────────────────────────────────────┐
│ ● Agent 就绪 │ ● OCR 已连接 │ ● LLM: GPT-4o │ 2/3   │
└──────────────────────────────────────────────────────┘
```

| 指示器 | 状态值 | 颜色规则 |
|--------|--------|---------|
| Agent | 就绪 / 执行中 / 错误 | 绿 / 蓝(脉冲) / 红 |
| OCR | 已连接 / 断开 | 绿 / 灰 |
| LLM | 模型名 + 可用状态 | 绿 / 红 |
| 并发 | 运行数 / 上限 | 灰色 |

### 2.3 子页面导航规则

所有子页面顶部统一显示面包屑：

```
← 首页 / 当前页面
```

点击 `← 首页` 返回首页，多级页面逐级展示（如 `← 首页 / 任务管理 / task_a3f8c2 / 监控`）。

### 2. 创建脚本 `/script/new` 和 `/script/:id/edit`

AI 对话式脚本创建/编辑。左侧对话交互，右侧 Markdown 实时预览。

- `/script/new`：创建新脚本，进入后显示空白对话区
- `/script/:id/edit`：编辑已有脚本，自动加载脚本数据并显示 Markdown 预览
- 顶部面包屑：`← 首页 / 脚本库 / 创建脚本` 或 `← 首页 / 脚本库 / 编辑: {脚本名}`
- 右上角"保存脚本"按钮（编辑模式显示"更新脚本"）
- 通过面包屑中的"脚本库"链接返回列表


**左侧面板 (≈60%)：**

| 区域 | 说明 |
|------|------|
| AI 对话区 | 聊天式交互，ScrollArea 自动滚动到底部 |
| 输入框 | Textarea, Enter 发送, Shift+Enter 换行 |
| 绑定窗口 | Select 下拉选择系统窗口 (HWND) |
| 脚本 | Radio: `○ 新建脚本` / `○ 选择已有脚本 ▾` |
| 模型选择器 | Select 切换 LLM 模型，实时生效 |

**右侧面板 (≈40%)：**

| 区域 | 说明 |
|------|------|
| MD 预览 | 渲染 Agent 生成的脚本内容 (Markdown) |
| 内容 | 脚本名、目标应用、变量、步骤列表 (含置信度) |
| 同步 | 对话中每次修改后右侧实时更新 |

**最小尺寸 (800×600)：** 左右分栏改为上下堆叠，预览区在下方可滚动。

```
│  │ 📄 数据录入 CRM                                   ││
│  │    将 Excel 客户信息逐条录入到网页 CRM 系统         ││

│  │                                    ▶ ✏ 📋 🗑   ││
│  ├──────────────────────────────────────────────────┤│
│  │ 📄 微信消息汇总                                   ││

```

| 元素 | 说明 |
|------|------|
| [+ 新建脚本] | 按钮，跳转到 `/script/new` |

| 📋 复制 | 克隆脚本，名称加"(副本)"后缀 |
| 🗑 删除 | 从列表中移除脚本 (带 Toast 确认提示) |
| 点击行 | 跳转到编辑页 `/script/{id}/edit` |

